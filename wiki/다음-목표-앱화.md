# 다음 목표 — 앱화 (출시)

[[콕콕콕]] 을 모바일 **앱으로 출시**하기. 현재는 React 웹앱(배포: `project-map-zeta.vercel.app`).

## 큰 그림 우선순위 (사용자 계획)

1. 로그인 기능 (✅ 완료 — [[기능#로그인]])
2. **앱 만들어서 출시** ← 지금 여기
3. 광고 달기

## 방식 선택 (핵심 결정)

지금 코드가 React 웹앱이라, 앱화는 크게 3가지:

| 방식 | 설명 | 장단 |
|---|---|---|
| **A. Capacitor (웹 감싸기)** ⭐추천 | 지금 웹 코드를 그대로 iOS/Android 네이티브 껍데기에 | 코드 재활용 최대, 스토어 출시 가능. WebView라 순수 네이티브보단 느낌 덜함 |
| **B. PWA** | 설치형 웹앱(홈화면 추가) | 가장 쉬움, 스토어 없이. 단 iOS 제약·"앱 같은 느낌" 약함 |
| **C. 네이티브 재작성 (RN/Flutter)** | 처음부터 다시 | 최고 UX, 그러나 재작성 비용 큼 |

→ **A(Capacitor)** 가 현실적. 지금 자산(지도·검색·AI코스) 그대로 쓰면서 스토어 출시.

## Capacitor로 갈 때 할 일 (개요)

- [ ] Capacitor 설치 + iOS/Android 플랫폼 추가, Vite 빌드(`dist`)를 웹뷰 소스로
- [ ] **OAuth를 앱용으로** — 웹 도메인 대신 **커스텀 스킴 딥링크**(예: `kokkokkok://auth-callback`)
  - Supabase Redirect URLs에 딥링크 추가
  - 구글: Android 클라(패키지명+SHA-1) / iOS 클라(번들ID)
  - 카카오: 네이티브 앱키 + 키해시(안드) / 번들ID(iOS), 카카오SDK 또는 커스텀 스킴
  - 네이버: 앱 콜백 스킴 (`/api/auth/naver/callback` 대신 딥링크로 복귀)
  - → **모바일 OAuth는 웹이랑 흐름이 달라서 별도 설정 필요** (지금 게이팅/세션 로직은 재활용)
- [ ] 위치 권한(내 위치), 상태바/스플래시/아이콘
- [ ] 스토어 등록 정보 — 소개문구(서비스 소개 초안 있음), 스크린샷, 카테고리(생활/여행), 개인정보처리방침
- [ ] 네이버 로그인 **검수 요청**(일반 공개용) — [[네이버-검수]] 준비 필요

## 미리 정할 것

- 앱 이름/번들 식별자 (예: `com.kokkokkok.app`)
- 딥링크 스킴
- 배포 도메인 유지 여부 (앱은 딥링크, 웹은 vercel 병행 가능)

## 참고

- 로그인 게이팅·세션·즐겨찾기 동기화 로직은 **그대로 재활용** — 앱화에서 바뀌는 건 주로 **OAuth 리다이렉트 방식**과 네이티브 셸.
- 광고(3번)는 앱 출시 후 AdMob 등으로.
