# 기능

[[콕콕콕]] 의 주요 기능 상세.

## 카테고리 (kind)

`전체 / 음식(food) / 관광지(travel) / 숙소(stay)` — `src/data/kinds.js` 중앙 설정.
카카오 카테고리 코드로 매핑: 음식=FD6, 관광지=AT4, 숙소=AD5. `전체`는 셋을 라운드로빈으로 섞음.
종류별로 **태그·키워드 칩**이 다름 (음식=고기/횟집/…, 관광지=궁궐/박물관/…, 숙소=호텔/펜션/…).

## 검색

- **자동완성** (`/api/suggest`, 카카오, 무료)
  - "근처/주변 + 키워드" → 지금 보는 지도 중심 반경검색
  - "지역 + 키워드" (개봉 전집) → 지역 지오코딩 후 그 동네에서 검색
  - 가게이름/브랜드 → 전국, 카테고리 넘어 재시도
- **엔터/검색** (`runTextSearch`) — 자동완성과 통일
  - 지역/근처 + 키워드 → **구글 텍스트검색**(장르 정확도↑, 카카오 느슨매칭 보완) + **주소 필터**(개봉이면 개봉동만)
  - 가게이름 → 전국 구글
  - 전국검색 시 지도가 결과 위치로 이동
- 왜 구글? — 카카오 키워드는 "전집" 검색에 냉면·부대찌개가 섞임. 구글 텍스트검색은 의미를 이해해 정확. 단 **엔터 때만** 호출해 비용 최소화.

## 지도 / 지구본

- 지구본(d3 orthographic) → 한국 핀 클릭/확대 시 Leaflet 지도로 전환, 축소하면 지구본 복귀 (`GeoPanel`)
- 지도 위 카테고리 버튼 "이 지역에서 찾기"(전체/음식/관광지/숙소)
- 내 위치 — 파란 점이 패널에 안 가리게 **오프셋**(웹=오른쪽, 모바일=위쪽)
- 대한민국 진입 포커스는 전국 중심(위도 35.9)

## 태그 (네이버 블로그 기반)

- `scripts/buildSeed.mjs` 로 동네별 카카오 수집 → **네이버 블로그 검색량**으로 태그 판정
- "동+이름" 정확매칭 비율로 노포·핫플·혼밥 등, 지역 상위 30% = 맛집/인기
- 구글 리뷰 안 씀(비용) → 태그=네이버, 별점·사진=구글
- 결과는 Supabase `seed` 테이블에 좌표+태그+블로그수 저장 (종류별 `kind` 컬럼)

## AI 하루 코스

- 진입: 헤더 **🧭 AI 코스** (→ [[기능#로그인]] 필요, 비로그인 시 로그인 유도)
- 런처 3모드: **지금 보는 지역 / ⭐즐겨찾기로 / 동네 선택** + 테마(자유입력)
- `/api/course`: 그 영역 seed 후보 → Gemini가 선택·순서·이유·제목 (id만 반환) → 우리 데이터 복원
- **구간 이동시간**: 🚗차(카카오 모빌리티), 🚶도보(거리 추정), 🚌대중교통(ODsay 키 있으면)
- 즐겨찾기 모드는 저장한 곳만, 없으면 근처 seed 병합 가능
- 키 없거나 실패 시 휴리스틱(블로그수+최근접) 폴백. Gemini 3회 재시도.
- 모델: `gemini-2.5-flash`

## 로그인 (소셜)

- **구글·카카오** — Supabase Auth 네이티브
- **네이버** — Supabase 미지원 → **커스텀 OAuth** (`/api/auth/naver/[action]`): code→토큰→프로필→Supabase admin 유저생성→magiclink 세션발급
- `src/lib/auth.jsx` (AuthProvider/useAuth), `LoginModal`, 헤더 아바타·드롭다운
- AI 코스는 로그인 유저만 (`Home.jsx` 게이팅)
- 카카오 이메일은 비즈앱 필요 → 비즈앱 전환함. 네이버 이메일 미사용(합성 이메일 처리)

## 즐겨찾기

- **게스트** → localStorage
- **로그인** → Supabase `favorites` 테이블 (RLS로 본인 것만)
- **로그인 순간** → 게스트 즐겨찾기를 계정으로 자동 병합 후 로컬 비움 (`mergeLocalFavorites`)
- 지도에 항상 별표시, `/saved` 페이지에서 모아보기
