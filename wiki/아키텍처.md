# 아키텍처

[[콕콕콕]] 의 기술 구조.

## 스택

| 레이어 | 기술 |
|---|---|
| 프론트 | React 18, Vite, React Router |
| 지도 | react-leaflet + Leaflet, MapLibre GL(벡터 basemap), d3-geo(지구본) |
| 서버 | Vercel 서버리스 함수 (`/api/*.js`), 로컬은 vite 플러그인이 동일 실행 |
| DB/인증 | Supabase (Postgres + Auth + RLS) |
| 외부 API | 카카오 로컬·모빌리티, 구글 Places(New), 네이버 블로그·로그인, Gemini |

## 폴더 구조

```
api/            서버리스 함수 (각 파일 = 1 함수, Vercel Hobby 한도 12)
  auth/naver/[action].js   네이버 로그인(동적 라우트, login·callback 1함수)
lib/            서버 공유 모듈 (함수 아님) — store.js, gcache.js
src/
  pages/        Home.jsx(메인), Saved.jsx
  components/   MapView, GeoPanel, Globe, FilterBar, CoursePanel,
                CourseLauncher, LoginModal, Header, DetailModal, RestaurantList
  lib/          places.js(검색 클라), supabase.js(즐겨찾기·인증클라), auth.jsx(AuthProvider)
  data/         kinds.js, countries.js, seed.json(로컬폴백) 등
scripts/        buildSeed.mjs (시드 빌더)
wiki/           이 위키 (Obsidian 볼트)
```

> `lib/` 를 `api/` 밖에 둔 이유: Vercel은 `api/` 안의 `.js`를 전부 함수로 세니, 공유 모듈을 밖으로 빼서 **함수 개수(12 한도)** 를 아낀다.

## 서버리스 함수 (11개, 한도 12)

| 함수 | 역할 |
|---|---|
| `kakao` | 카카오 로컬 검색(영역/키워드, 격자수집) + 구글 보강 |
| `places` | 구글 Places 텍스트검색(전국·장르검색) |
| `place` | 구글 장소 상세(사진·영업시간) |
| `photo` | 구글 사진 프록시 |
| `suggest` | 검색 자동완성(지역+키워드/근처 인식) |
| `geocode` | 카카오 주소→좌표 |
| `seed` | 저장된 태그 장소 조회(태그 필터) |
| `curation` | 화제의 맛집 큐레이션 |
| `course` | AI 하루 코스(Gemini + 이동시간) |
| `seo` | /place/:id, sitemap, robots (SEO) |
| `auth/naver/[action]` | 네이버 로그인 login·callback |

## 데이터 흐름 (검색)

```
사용자 입력 → suggest(자동완성, 카카오)
엔터/검색 → runTextSearch (Home.jsx)
   ├ "근처 X"     → 현재 지도영역 구글 텍스트검색
   ├ "지역 X"     → 지역 지오코딩 → 그 영역 구글 텍스트검색 + 주소필터
   └ 가게이름     → 전국 구글 검색
지도 카테고리버튼 → 현재 영역 카카오 검색(음식/관광지/숙소)
결과 → 구글 별점·사진 보강(gcache 캐시) → 리스트+지도 마커
```

## 데이터 흐름 (AI 코스) — [[기능#AI 하루 코스]]

```
헤더 🧭 AI 코스 (로그인 필요)
 → 런처: 위치(현재/즐겨찾기/동네) + 테마
 → /api/course : seed에서 그 영역 후보 수집(+즐겨찾기 병합)
 → Gemini가 고르기·순서·이유·제목 (id만 반환 = 환각방지)
 → 우리 데이터로 복원 + 구글 보강 + 구간 이동시간(카카오 모빌리티/도보추정)
 → CoursePanel(왼쪽) + 지도 번호핀·점선
```

## 저장 계층 (`lib/store.js`)

- `SUPABASE_URL` + `SUPABASE_SERVICE_ROLE` 있으면 **Supabase**, 없으면 **로컬 파일**(`src/data/seed.json`) 폴백
- 테이블: `seed`(태그 장소), `seed_scanned`(분석한 지역), `gcache`(구글 캐시), `favorites`(즐겨찾기)
