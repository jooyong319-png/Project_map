# 배포와 키

[[콕콕콕]] 배포 · 환경변수 · OAuth 설정. **시크릿 값은 여기 안 적음** — 값은 로컬 `.env`(gitignore)와 Vercel/Supabase 대시보드에.

## 배포 (Vercel)

- main 브랜치 push → 자동 빌드
- 도메인: `project-map-zeta.vercel.app`
- 서버리스 함수 **11개** (Hobby 한도 12 — 여유 1). 함수 추가 시 통합 고려.
- `vercel.json`: `/place/:id`·sitemap·robots 리라이트 + SPA 폴백

## 환경변수 (Vercel Settings → Environment Variables, 11개)

값은 `.env`에서 복사 (Vercel은 `.env` 통째 붙여넣기 파싱 지원). `VITE_` 2개만 브라우저 노출용.

| 키 | 용도 |
|---|---|
| `GOOGLE_PLACES_API_KEY` | 구글 별점·사진·장르검색 |
| `KAKAO_REST_KEY` | 카카오 로컬 검색 (지도용, 로그인앱과 별개) |
| `NAVER_CLIENT_ID` / `NAVER_CLIENT_SECRET` | 네이버 **블로그** 검색(태그) |
| `NAVER_LOGIN_CLIENT_ID` / `NAVER_LOGIN_CLIENT_SECRET` | 네이버 **로그인**(별개!) |
| `SUPABASE_URL` / `SUPABASE_SERVICE_ROLE` | DB·admin (서버 전용) |
| `GEMINI_API_KEY` | AI 코스 |
| `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` | 브라우저 인증 클라 |
| `ODSAY_API_KEY` | (선택) 대중교통 이동시간 |

> ⚠️ 환경변수 넣은 뒤 **재배포**해야 적용됨.
> ⚠️ 네이버는 `NAVER_CLIENT_*`(블로그)와 `NAVER_LOGIN_CLIENT_*`(로그인) **4개** — 헷갈리지 말 것.

## OAuth 콜백 (프로바이더별)

공통 원칙: 구글·카카오의 **Redirect URI = Supabase 콜백** (환경 무관, 안 바뀜):
```
https://wkoonnyitfvzxjclqhph.supabase.co/auth/v1/callback
```
Vercel 도메인은 "로그인 후 복귀 주소" 쪽에만 들어감.

| 서비스 | 넣는 곳 | 값 |
|---|---|---|
| 카카오 | 카카오 로그인 → **Redirect URI** | Supabase 콜백 (위) |
| 카카오 | 플랫폼 Web 사이트도메인 | localhost + `https://project-map-zeta.vercel.app` |
| 구글 | OAuth 클라 → **승인된 리디렉션 URI** | Supabase 콜백 (위) |
| 구글 | 승인된 JavaScript 원본 | localhost + vercel |
| 네이버 | 로그인 환경 → **Callback URL**(여러개) | `http://localhost:5173/api/auth/naver/callback` + `https://project-map-zeta.vercel.app/api/auth/naver/callback` |
| 네이버 | 서비스 URL(1개) | 배포 도메인 (또는 로컬/배포 앱 분리) |

## Supabase 설정

- **Authentication → Providers**: Google·Kakao 활성화 + Client ID/Secret. 카카오는 "Allow users without email" ON.
- **Authentication → URL Configuration**:
  - Site URL: `https://project-map-zeta.vercel.app`
  - Redirect URLs: `http://localhost:5173/**` + `https://project-map-zeta.vercel.app/**`
- **SQL**: `favorites` 테이블 + RLS (본인 것만), `seed`/`seed_scanned`/`gcache`에 `kind` 컬럼

## 삽질 로그 (겪은 에러)

- 카카오 KOE205 = 이메일 동의 미설정 → 비즈앱 전환
- 카카오 KOE006 / 구글 redirect_uri_mismatch / 네이버 disp_stat=207 = **콜백 URI 미등록** (앱 혼동·칸 착각 주의)
- 네이버 "개발중 상태" = 등록 아이디만 → 소유자 계정 테스트 or [[네이버-검수]]
- 로그인 후 localhost로 튐 = Supabase Redirect URLs에 프로덕션 도메인 누락
- Supabase 프로젝트 미사용 자동 일시정지 → Restore
- 헤더 `backdrop-filter` 때문에 모달 `position:fixed`가 헤더 기준 → body 포털로 해결
